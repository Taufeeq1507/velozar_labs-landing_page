NIS2 Compliance Monitor for Jira

OFFICIAL DOCUMENTATION · V1.0 · FOR JIRA CLOUD

Welcome to the NIS2 Compliance Monitor for Jira. This enterprise-grade Governance, Risk, and Compliance (GRC) tool automatically detects potential cyber incidents in your Jira workflows, enforces strict internal reporting SLAs, and generates immutable audit logs to help your organization comply with Directive (EU) 2022/2555 (NIS2 Article 23).

1. Getting Started (For Jira Admins)

To prevent alert fatigue and restrict scanning to relevant teams, Jira Administrators must configure the application scope before use.

Accessing Global Settings

  • In the top Jira navigation bar, click Apps > NIS2 Compliance.
  • Select the Configuration tab.

Recommended Configuration

  • Trigger Keywords: Enter comma-separated terms that indicate a security incident (e.g., breach, ransomware, data leak, unauthorized access).
  • SLA Deadline: Define your internal reporting countdown (Default: 24 hours).
  • Monitored Projects: Restrict the app to specific departments by entering Project Keys (e.g., SEC, IT, LEGAL). Leave blank to monitor all projects.
  • Monitored Issue Types: Restrict scanning to specific workflows (e.g., Incident, Security).

2. Automated Detection & Alerting

The app operates silently in the background, scanning the Summary and Description fields of new and updated issues within your configured scope.

The Anti-Alert Fatigue Failsafe

We know developer workflows can be noisy. The app uses contextual intelligence to categorize alerts based on Jira Priority:

  • Breach Detected High / Highest Priority + Keyword: Instantly triggers a critical SLA countdown and posts a warning comment in the Jira thread.
  • Review Required Low Priority + Keyword: Silently flags the issue for review on the CISO Dashboard without spamming the issue thread with comments.

3. Triaging & Submitting Reports

When an issue triggers a breach alert, Security and Compliance teams use the native Jira Issue Panel to manage the response.

Using the Issue Panel

  1. Open the flagged Jira issue. On the right-hand sidebar, locate the NIS2 Compliance panel.
  2. Observe the real-time SLA countdown timer.
  3. Click Prepare Compliance Payload to open the assessment form.
  4. Fill in the incident details (Jurisdiction, Entity Name, Cross-Border Impact).
  5. Click Auto-Attach & Close.

What happens next? The app generates an official PDF compliance report, attaches it securely to the Jira issue, stops the SLA timer, and posts an immutable Audit Log comment recording the user who authorized the submission.

4. The CISO War-Room Dashboard

The centralized Dashboard provides a top-down view of all compliance activities across your organization.

  • Overdue The 24-hour internal SLA has expired. Immediate escalation required.
  • Pending Active breach. The countdown timer is currently running.
  • Review Low-priority failsafe triggered. Requires a manual check to dismiss or escalate.
  • Compliant The incident was successfully processed and the PDF report is attached.

5. Frequently Asked Questions (FAQ)

Does Velozar Labs have access to our Jira data?

Absolutely not. This app is built on the secure Atlassian Forge architecture. It has a strict Zero-Egress policy. Your Jira data is never transmitted to, processed by, or stored on Velozar Labs servers. All background processing happens natively inside your Atlassian Cloud environment.

Why didn't an issue trigger an alert?

If an issue contains a keyword but didn't trigger an alert, verify that the issue's Project Key and Issue Type are included in the Monitored Projects/Types list in your Global Configuration.

Can we export the Dashboard data?

Because the app applies native Jira Labels (e.g., nis2-breach, nis2-compliant), you can easily export this data using standard Jira JQL searches:
labels in (nis2-breach, nis2-review-required) ORDER BY created DESC.
